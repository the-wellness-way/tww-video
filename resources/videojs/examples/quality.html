HLS or MPEG/DASH resolutions


Resolution qualities are detected automatically for HLS or MPEG/DASH video streams if only HLS or DASH protocol includes links to multiple separate resolution playlists. 

<!--HLS video on page setup-->
<video id="player_1" class="video-js vjs-fluid" controls playsinline poster="../examples/assets/images/people.jpg">
	<source src="//opencdn.b-cdn.net/video/hls/people/playlist.m3u8" type="application/x-mpegURL">
</video>

<!--Initialize player and plugin-->
<script>
	var player1 = videojs('player_1');
	player1.nuevo({singlePlay:true});
</script>
Qualities menu resolution and bitrate values. You can limit it to show only resolution using appropriate option. The option will not affect if there are more than one bitrate playlist for the same resolution. 
<script>
	var player1 = videojs('playe');
	player1.nuevo({ resOnly: true });
</script>

Showing Qualities menu as Settings submenu


By default qualities menu is displayed as separate menu in control bar. You can also order to display it as setting submenu by setting nuevo plugin option qualityMenu: true. 
<script>
	var player4 = videojs('player_3');
	player4.nuevo({ qualityMenu: true });
</script>

The HD or 4K icon appears for videos of minimum 1080p resolution. This is what Youtube apply as well.
HD icon for resolution > 720 hase been depreciated, however the user may still decide about the minimum resolution to show HD icon for. 
<script>
	var player = videojs('player_1');
	player.nuevo({ 
	    minhd: 720
	});
</script>

Note: Each instance of the player with the nuevo plugin make use of singlePlay enabled option. This allows to play only one player at the same time and pause others automatically.
